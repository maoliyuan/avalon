# 阿瓦隆记牌器

一个**纯前端、可离线**的阿瓦隆私人记牌 / 身份推理工具。为「线下只有一台 iPhone」的场景做的：装到主屏当独立 App 用，数据只存在你自己手机本地。

## 功能

- **身份标记（核心）**：对每个号码（1–12）记录身份。每条 = `确定度 × 身份`，五档确定度：
  - **最终是**（揭底后的真实身份，金色，作为分析用的「标准答案」）/ 确定是 / 可能是 / 可能不是 / 确定不是
  - **可叠加**：同一号码可挂多条（如 3 号：`可能是梅林` + `确定不是坏人`）。点标记即删。
  - 可标记的身份清单在 [`app.js`](app.js) 顶部的 `IDENTITIES` 数组里，随手增删。
- **每轮记录**：每轮记下 队长 / 提名队员 / 每个号码的赞成·反对。纯客观记录，**不做任何推算，不给你 bias**。
- **本地存储 + 离线**：首次联网打开一次后即可全程离线；关掉重开数据还在。
- **新开一局 / 回溯上一局**：新开一局会把当前记录存为「上一局」；「回溯上一局」在这一局和上一局之间来回切换（只保留一层）。

## 真实对局记录 & 导出（喂给分析 workflow）

为了事后分析「常玩对手的决策」，可以把**真实**对局完整记录并导出成 JSON：

- **真实玩家名册（可跟不同的人玩）**：在「结果 / 导出」页点「管理名册」，登记真实的人（每人一个稳定 `id`，可改名 / 删除）；也可在指派座位时直接「＋ 新增真实玩家」。名册**跨局持久保存**，换一批人只需再登记新的人。
- **座位指派**：在「身份标记」页每个号码卡上点「指派玩家」，把座位号绑定到某个真人。同一人一局只占一个座位。**座位会跨局沿用**：新开一局不再要求重指，同一天的多局都用同一套座位；**每天北京时间 0 点自动重置**（换新的一天/新的一桌人时从零开始）。名册本身始终保留，不受重置影响。
- **真实身份**：用身份标记里的 **「最终是」**（金色）标注揭底后的真实角色 —— 这就是分析用的标准答案标签。
- **对局结果**：「结果 / 导出」页记录 胜负 / 每个任务成败 + 失败票数 / 刺杀目标与是否刺中梅林。
- **导出**：
  - 「导出本局」= 当前这一局，一个 JSON 文件；
  - 「导出全部（含存档）」= 当前 + 上一局 + 历史存档合并成一个 JSON（每次「新开一局」自动按对局 `id` 去重存档，最多留 100 局）。
  - iPhone 上优先弹**系统分享菜单**（存到「文件」App / AirDrop / 发给其它 App），不支持时退回普通下载。
- **导出 JSON 结构**（`formatVersion: 1`）：每个 `game` 含
  - `players[]`：`seat`（座位号）、`person`（`{id,name}` 或 null）、`finalIdentities`（「最终是」真实身份）、`guesses`（主观猜测）、`actions`（`proposalsAsLeader` 当队长时的提名、`votes` 每轮投票）；
  - `rounds[]`：`leader` / `team` / `votes`；
  - `result`：`winner` / `missions[]`（成败+失败票）/ `assassin`（`target` + `hitMerlin`）。

## 实时助手（配合本地 `avalon_supporter`）

「结果 / 助手」页和「助手」页可以把对局实时发给你电脑上的本地 workflow（用 Opus 4.8 分析），结论回钉钉、聊天回手机。上行/下行走 **ntfy 轮换频道**（频道名由密钥+时间派生、每 6h 自动换，陌生人猜不到，不易被打扰）。

- **频道设置**（「助手」页 →「频道设置」）：填一个**频道密钥**，和电脑端 `AVALON_CHANNEL_SECRET` 填成**同一个**即可（可点「随机生成」）。手机什么都不用装，只是发 HTTPS 请求。
- **本局配置**：坏人数（默认按人数，可改）+ 在场特殊角色。
- **我的身份**：我坐几号 + 我的角色 —— 选了角色后会出对应的**夜间情报**标记：梅林→标看到的坏人；派西→标 2 个拇指；坏人→标坏队友。这些是「亲眼所见的硬事实」，分析最看重。
- **用助手分析**：把当前整局 + 配置 + 我的身份 + 夜间情报打包发到本地 → 电脑分析后发**钉钉**。
- **发送本局到本地（导出）**：一整局结束把整局直接发给本地存档（取代文件下载，省事）。
- **复盘**：填一整局复盘文字，「发送复盘到本地」并入该局存档，供以后分析。
- **聊天**（「助手」页）：发一句话问助手，电脑用 Opus 回答后**回传手机**，在消息区显示。

> 需要电脑端跑起 `avalon_supporter`（见其 README）。载荷过大时自动 gzip；两端频道派生算法一致（已跨语言校验）。

## 部署到 GitHub Pages

1. 新建一个仓库（比如叫 `avalon`），把本目录所有文件传上去（`index.html` 必须在仓库根目录）。
2. 仓库 **Settings → Pages → Build and deployment**：Source 选 `Deploy from a branch`，分支选 `main`、目录选 `/ (root)`，保存。
3. 等一两分钟，页面地址是 `https://<你的用户名>.github.io/<仓库名>/`。

> 全部用相对路径写的，放在子路径（`/仓库名/`）下也能正常离线，无需改动。

## iPhone 上安装

1. 用 **Safari** 打开上面的地址（必须 Safari，Chrome 不行）。
2. 点底部「分享」→ **添加到主屏幕**。
3. 之后从主屏图标打开：全屏、无地址栏，跟原生 App 一样，且**可离线使用**。

数据存于该网站的本地存储中。装到主屏后系统一般会长期保留；只要别在设置里清除该网站数据或删掉图标，记录就一直在。

## 更新版本

改完代码后，把 [`sw.js`](sw.js) 里的 `VERSION` 加一（`v1` → `v2`…）再推送。各设备下次联网打开时会自动拉到新版。

## 设置 / 修改密码

打开时有一道密码锁（防止路人乱点；**注意这是纯客户端校验，只挡随手党，不是强安全**）。输入正确密码后 **8 小时内**免再输，也可点顶栏「🔒 锁定」立即上锁。

密码不是明文存的，存的是它的 **SHA-256 哈希**。改密码步骤：

1. 生成新密码的哈希（Mac 终端）：
   ```bash
   printf '你的新密码' | shasum -a 256
   ```
   取输出最前面那串 **64 位十六进制**（别带后面的空格和 `-`）。
2. 把它填到 [`app.js`](app.js) 顶部的 `PASSWORD_HASH = '...'`。
3. 把 [`sw.js`](sw.js) 的 `VERSION` 加一，提交推送即可。

> 当前预置的是临时密码 `avalon`，务必改掉。
